# Contribuindo

1. Crie uma branch curta a partir de `main`.
2. Faça mudanças pequenas e commits Conventional Commits.
3. Execute `npm run validate` e `npm run format:check`.
4. Inclua testes para regras financeiras e regressões.
5. Abra um pull request draft descrevendo impacto, riscos e evidências.

Não inclua `.env`, service accounts, dados reais de usuários ou alterações do caminho `/cfp-money/` sem plano de migração.
